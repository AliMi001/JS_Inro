• API, ça veut dire Application Programming interface (interface de programmation en français). Et ça va être ce qu'on a au milieu.
  • C'est une interface qui nous permet d'aller piocher des données dans une base de données
  • get post put et delete: des requêtes HTTP (On utilise beaucoup GET qui nous permet de prendre des données directement.)
  • Une fois que ces données vont revenir, elles vont arriver en deux formats, même s'il y a un format qui a supplanté l'autre dernièrement, c'est le format JSON (JavaScript Object notation) ici. (Et on a également le format XML)
  • Une API: c'est juste une interface avec laquelle on va discuter pour aller chercher des données.

Qu'est ce que le format JSON?

  • c'est un format de fichier qui permet de stocker facilement les données.
  • Ça a été créé vraiment pour manipuler des données, pour pouvoir les recevoir et en faire ce qu'on veut facilement en fait.
  • Et donc une des grandes des grands changements entre du jason et du javascript: les propriétés vont être entre guillemets.
    Et aussi qu'on démarre un fichier, disons comme un objet englobé d'une accolade ouvrante en haut et d'une accolade fermant en bas.
